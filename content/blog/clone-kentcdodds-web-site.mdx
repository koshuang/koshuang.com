## 緣起

最近開始想建立個人部落格，因此先去 Godaddy 註冊了 koshuang.com (目前還是 Godaddy 預設頁面）。

在左思右想要使用什麼平台的時候，突然想到 Kent C Dodds 他把自己的部落格放到 GitHub 上，並使用 Remix 改寫。

所以我也在想，如果把他的 Repo Clone 下來改成自己的，會如何？

目前想到幾個優點：

👉 Kent 把他的個人部落格當成 Open Source 這個概念很酷（留言處我會放一篇他的文章，關於他如何建置網站，內容超豐富！）

👉可以 Open Source，也許哪天有讀者有興趣，也可以來幫忙修正錯誤 XD

👉可以趁機學 Remix

👉可以學 Kent 他怎麼架構一個網站

所以這篇文章是想要紀錄我在 Clone 過程中做了些什麼事，以及心得。所以會以日記的方式來呈現，結構可能會比較鬆散。
之後也會持續更新日記。

## 4/3

- Clone repo
- 設定基礎環境: `npm run setup`，他會執行任務
  - 安裝 `npm install`
  - 建立 sqlite & cache files
  - Preload DB seeds
  - 執行驗證與測試，確保安裝與環境設定沒有問題

## 4/12

- 把一些我不需要的內容移除
  - `content/blog` 下複製一個原本的文章來當範本，建立這篇文章
  - 刪除其他的部落格文章
  - 移除部分 Talks 的內容
- Rename
  - 把他的名字換成我的名字
  - 更改網站 domain
- 遇到的問題
  - 刪除文章後，Cache 似乎不會刪除
  - 他疑似有做 Browser cache，有時候刷新之後又出現舊的內容
  - 更改分類和關鍵字，他好像也不會自動更新，或是需要等
- 解法
  - 後來都是手動去把 sqlite & cache 清除再重新跑一次 setup 來解決

## 4/13

主要進行中文化，並思考文案。

- 整理之前的 WIP Git commit
- 更改 Navbar
  - 只留下自我介紹跟部落格，未來累積更多內容之後再開放

## 4/14

- 調整首頁
  - 持續進行中文化，關於部落格說明以及我想提供的服務，調整文案。
  - 移除還無法提供的服務
- 部署目前的內容到 koshuang.com
  - 為了節省成本，先找一台在 AWS 原本已經有 Host 的機器
  - 安裝 nvm，並安裝 Node.js 18
  - 在 Global 安裝 pm2: `npm install -g pm2`
  - 安裝專案
    - `npm run setup`
    - `npm build`
      - 遇到的問題
        - better-sqlite3 一直 build 失敗
          - 系統需要安裝 sqlite3 與 gcc,
            - `apt-get install build-essential sqlite3 gcc`
    - 後來發現他的 Dockerfile 有很多東西，感覺直接用 Docker 執行可能會比較好。
      - 遇到的問題
        - server 上 docker build 一直在 npm install 時失敗，之後有時間再戰

反饋

- 為了加速每次改文案，因次我盡量使用 ChatGPT 產生內容，或是我在 Threads 的內容。
- 先求有，再求好。
  - 以 DevOps 的部署左移概念，應該要盡量在前期就讓網站部署上去，即使只有空的內容。
  - 為達成上面目的，主要目標還是希望盡快的中文化，只留下我自己的資訊（避免抄襲或是侵犯到 Kent 的智慧財產權），讓網站先上線。
- 在 Server 上遇到的問題
  - 其實 Kent 有提供 Dockerfile，只是我在安裝遇到問題時，應該要直接去參考 Dockerfile，至少要安裝裡面全部的套件，應該可以縮短更多處理錯誤的時間。
